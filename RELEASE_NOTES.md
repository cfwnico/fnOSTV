# 发布版本信息

## 当前开发版

- 包名：`com.fnostv.android4`
- 当前版本号：`0.1.0`
- Version Code：`1`
- 最低系统：Android 4.2 / API 17
- 目标系统：Android 9 / API 28
- 当前主线：原生轻量客户端

## 已实现能力

- 原生登录和会话恢复。
- fnOS RPC/WebSocket 连接。
- 原生首页、媒体库入口、分类入口、收藏入口和最近播放。
- 本地媒体库管理：添加、编辑、删除和扫描媒体库目录。
- 本地媒体索引：扫描结果用于首页、影视大全、分类和搜索。
- 影视大全稳定加载：REST 影视服务优先，本地索引兜底，再回退 RPC mediaCenter 和文件模式。
- 文件库目录浏览和媒体文件识别。
- 收藏、最近播放本地存储。
- 首页海报墙分区：继续观看、收藏和影视大全按有限数量展示，避免旧设备一次渲染过多卡片。
- 影视详情页：从媒体列表进入详情页后再播放，并尽量展示年份、评分、简介和剧集/版本数量。
- 播放源选择：支持播放前查看和切换可用播放源。
- 首次配置流程：首次保存有效服务器和账号配置后，自动返回主流程并继续原生登录。
- 遥控器增强：详情页支持菜单键收藏、播放键播放、返回键关闭。
- 原生播放器页，支持播放/暂停、进度条、快进/快退、倍速、画面适应/铺满、清晰度源切换。
- LibVLC 优先播放，IJKPlayer 兜底，最终回退外部播放器 Intent。
- 播放性能配置：按播放 URL 选择缓存档位，记录解码、缓存和缓冲日志。
- 旧设备性能优化：局域网/本机低风险 MP4 使用更快的低延迟启动档位，高风险格式继续保留流畅兜底策略。
- 海报解码采样保护：远程海报先读取尺寸再按目标尺寸采样，减少海报墙内存峰值。
- 飞牛影视风格的原生登录页、首页布局和线性小图标。
- 飞牛影视风格的原生设置页，包含账号连接和媒体库管理。
- Android 4.4 x86 本地模拟器安装调试脚本。
- debug/release 构建脚本。

## 播放器实现

当前播放抽象为：

```text
NativeVideoPlayerView
└── PlayerEngine
    ├── VlcPlayerEngine
    └── IjkPlayerEngine
```

默认策略：

- 优先创建 `VlcPlayerEngine`。
- VLC 失败后自动切换 `IjkPlayerEngine`。
- IJK 硬解失败后自动切换软解。
- 所有内置播放器失败后交给外部播放器。

当前默认缓存策略：

- HTTP/远程流：`networkCachingMs=6000`
- 稳定模式：`networkCachingMs=3000`
- 局域网/本机低风险 MP4：`networkCachingMs=2500`，`fileCachingMs=1000`
- 默认关闭跳帧/丢帧，减少画面不连续。

## 已知限制

- LibVLC 3.1.12 的 AAR 声明最低 API 17，因此默认构建不再覆盖 Android 4.0/4.1。
- 当前媒体库管理优先使用本地配置和文件扫描；影视大全已加入 REST、本地索引、RPC mediaCenter、文件模式的稳定回退链路。
- 原生海报墙仍需继续丰富内容分区，当前详情页已加入非阻塞的详情元数据兜底展示。
- 清晰度切换依赖 fnOS 返回多播放源；没有多码率源时只能显示当前源和实际分辨率。
- 不同旧电视盒子的硬解能力差异较大，遇到 H.265、10bit、高码率或特殊封装时仍可能需要 IJK 软解或外部播放器兜底。
